## 1. JWT 认证流程

注册采用 Admin 审批策略（`RegistrationPolicy` Protocol 实现），用户注册后进入 pending 状态，需要管理员审批后才能登录。第一个注册用户自动审批并成为管理员。RegistrationPolicy 使审批策略可替换，Phase 2 可切换为邀请码策略或开放注册。

审批后用户不自动获得 JWT，需通过 `POST /auth/login` 重新登录获取 token。审批端点只返回用户状态变更确认，不签发 JWT。

### 1.1 用户注册 POST /auth/register

**请求：**

```json
{
  "username": "string (3-32 chars, alphanumeric + underscore)",
  "password": "string (8+ chars)",
  "display_name": "string (1-64 chars)"
}
```

**响应 200（注册成功，等待审批）：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "display_name": "string",
  "status": "pending",
  "message": "Registration submitted, awaiting admin approval"
}
```

**响应 200（第一个用户，自动审批并返回 JWT）：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "display_name": "string",
  "status": "active",
  "access_token": "string (JWT)",
  "token_type": "bearer",
  "message": "Auto-approved as the first admin"
}
```

**错误响应：**

- 409 — 用户名已存在
- 格式：`{"detail": "Username already exists"}`

### 1.2 用户登录 POST /auth/login

**请求：**

```json
{
  "username": "string",
  "password": "string"
}
```

**响应 200（active 用户）：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "display_name": "string",
  "access_token": "string (JWT)",
  "token_type": "bearer"
}
```

登录响应不含 `is_admin` 字段（MVP 裁剪：前端通过解码 JWT payload 获取 `is_admin`，无需额外 API 调用）。

**错误响应：**

- 401 — 用户名或密码错误
- 403 — 用户状态为 pending（`{"detail": "Account pending admin approval"}`）
- 403 — 用户状态为 rejected（`{"detail": "Account registration was rejected"}`）

### 1.3 管理员审批用户 POST /auth/users/{id}/approve

**认证：** 必须，且 `is_admin=True`

**响应 200：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "display_name": "string",
  "status": "active"
}
```

**错误响应：**

- 403 — 当前用户不是管理员
- 404 — 用户不存在

### 1.4 管理员拒绝用户 POST /auth/users/{id}/reject

**认证：** 必须，且 `is_admin=True`

**请求（可选）：**

```json
{

```

**响应 200：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "status": "rejected",
  "reason": "string | null"
}
```

**错误响应：**

- 403 — 当前用户不是管理员
- 404 — 用户不存在
- 400 — 用户已经是 active 或 rejected 状态

### 1.8 RegistrationPolicy Protocol

注册策略通过 Protocol 抽象，使审批流程可替换：

```python
from app.hub.auth.repository import UserRepositoryProtocol

@runtime_checkable
class RegistrationPolicy(Protocol):
    async def evaluate(
        self,
        request: UserRegister,
        repo: UserRepositoryProtocol,
    ) -> RegistrationDecision: ...

class RegistrationDecision(BaseModel):
    status: Literal["pending", "active"]   # MVP: pending（首个用户 active）
    is_admin: bool = False                  # MVP: 仅第一个用户 True
    message: str = ""
```

`evaluate` 接收 `repo: UserRepositoryProtocol` 参数，MVP 的 AdminApprovalPolicy 通过 `repo.has_active_users()` 判断是否首个用户。Phase 2 替换为 InviteCodePolicy 时可通过 repo 的布尔方法实现（如 `repo.check_invite_code_exists()`），不再降低抽象级别直接操作数据库连接。

MVP 实现 `AdminApprovalPolicy`：检查 users 表是否为空（无 active 用户），第一个用户自动审批并设为 `is_admin=True`，后续用户一律 pending。Phase 2 可替换为 `InviteCodePolicy`（凭邀请码直接 active）或 `OpenRegistrationPolicy`（直接 active）。

### 1.9 JWT 结构

```json
{
  "sub": "user_id (UUID)",
  "username": "string",
  "display_name": "string",
  "is_admin": false,
  "iss": "orbion",
  "exp": 1706000000,
  "iat": 1705990000
}
```

JWT payload 不包含项目角色信息（项目角色通过 `GET /projects` 响应中的 role 字段获取，同一用户在不同项目有不同角色）。但包含 `is_admin` 平台级管理员标识和 `display_name`（避免每次请求都查数据库）。

### 1.10 JWT 配置

| 参数 | 值 | 说明 |
|------|---|------|
| 算法 | HS256 | MVP 用对称密钥，简单高效 |
| 密钥 | 环境变量 JWT_SECRET | 不硬编码，从 config 读取 |
| 过期时间 | 7 天 | 自我构建阶段不需要频繁登录 |
| 签发者 | "orbion" | JWT iss 字段 |

### 1.11 FastAPI 认证依赖

```python
# auth/dependencies.py
async def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    """从 JWT token 解析当前用户。

    所有需要认证的端点注入此 dependency。
    返回 User 对象（user_id, username, display_name, is_admin）。
    """
    payload = jwt.decode(token, config.jwt_secret, algorithms=["HS256"])
    user_id = payload["sub"]
    username = payload["username"]
    return User(id=user_id, username=username, display_name=payload.get("display_name", ""), is_admin=payload.get("is_admin", False))

async def require_admin(user: User = Depends(get_current_user)) -> User:
    """要求当前用户是管理员。审批端点注入此 dependency。"""
    if not user.is_admin:
        raise HTTPException(status_code=403, detail="Admin access required")
    return user
```

所有需要数据库操作的端点通过两层依赖注入获取 UserRepository：

```python
from app.hub.auth.repository import UserRepositoryProvider

def _get_user_repo_provider(request: Request) -> UserRepositoryProvider:
    """从app.state获取UserRepositoryProvider单例"""
    return request.app.state.user_repo_provider

# 路由端点使用模式
@router.post("/register")
async def register(
    request: UserRegister,
    provider: UserRepositoryProvider = Depends(_get_user_repo_provider),
):
    async with provider.scoped() as repo:  # per-request事务隔离
        await repo.check_username_exists(request.username)
        ...
```

Provider 是单例（app.state 生命周期），scoped() 为每个请求创建独立的事务 scoped 实例（async context manager yielding UserRepositoryProtocol），多个 CRUD 操作共享同一事务。

## 4. 消息 API

### 4.1 POST /threads/{id}/messages — 发送消息

**认证：** 必须，项目成员

**请求：**

```json
{
  "content": "string (1-10000 chars)",
  "request_summary": false
}
```

**响应 200：**

```json
{
  "id": "string (UUID)",
  "thread_id": "string",
  "participant_id": "string",
  "participant_type": "human",
  "display_name": "string",
  "content": "string",
  "created_at": "datetime"
}
```

**逻辑：**

1. 写入 thread_messages 投影
2. 写入 EventStore（DiscussionMessageCreated 事件）
3. 通过 EventBus.publish 路由
4. 如果 `request_summary=true`，EventBus 传递到总结 Agent（payload 中附加此标志）

**前端交互：** `request_summary` 标志通过斜杠命令触发——用户在讨论输入框输入 `/summarize`，前端解析后将 `request_summary` 设为 true 发送。斜杠命令列表：`/summarize`（请求总结）、`/help`（显示命令帮助）。

### 4.2 GET /threads/{id}/messages?before={msg_id}&limit=50 — 消息列表

**认证：** 必须，项目成员

**查询参数：**

- `before` — 游标分页，返回此 ID 之前的消息（逆序时间线）
- `limit` — 默认 50，最大 100

**响应 200：**

```json
[
  {
    "id": "string",
    "participant_id": "string",
    "participant_type": "human | agent",
    "display_name": "string",
    "content": "string",
    "event_type": "DiscussionMessageCreated | DiscussionSummaryGenerated | ExecutionPlanProposed",
    "created_at": "datetime"
  }
]
```

**逻辑：** 人类发言和 Agent 产出在同一条消息流中，通过 `participant_type` 和 `event_type` 区分样式。前端根据 `participant_type` 选择不同消息卡片样式（人类用普通样式，Agent 用特殊样式）。

- `status` — 可选，按状态过滤

```json

  {
    "id": "string",
    "thread_id": "string | null",
    "status": "proposed | approved | rejected | executing | completed",
    "proposed_by": "string (participant_id)",
    "tasks": [
      {
        "task_id": "string",
        "type": "code | document | configuration",
        "description": "string",
        "status": "pending | in_progress | completed",
        "dependencies": ["task_id"],
        "priority": "high | medium | low"
      }
    ],
    "created_at": "datetime"
  }
]
```

### 5.2 POST /plans/{id}/approve — 审批计划

**认证：** 必须，项目 Owner/Admin/Member（有 APPROVE_PLAN 权限位）

**请求：**

```json
{
  "approved_tasks": ["task_id1", "task_id2"],
  "modifications": {
    "task_id3": {
      "description": "修改后的描述"
    }
  }
}
```

**响应 200：**

```json
{
  "plan_id": "string",
  "status": "approved",
  "approved_tasks": ["task_id1", "task_id2"],
  "modifications": {}
}
```

**逻辑：**

1. 更新 execution_plans 投影状态
2. 写入 EventStore（ExecutionPlanApproved 事件）
3. 通过 EventBus 路由到执行 Agent
4. 未批准的任务不传递给执行 Agent

支持部分审批：只批准某些任务，对其他任务提供修改意见。

### 5.3 POST /plans/{id}/reject — 拒绝计划

**认证：** 同 approve

**请求：**

```json
{
  "reason": "string",
  "suggestions": ["string"]
}
```

**响应 200：**

```json
{
  "plan_id": "string",
  "status": "rejected",
  "reason": "string",
  "suggestions": ["string"]
}
```

**逻辑：**

1. 更新 execution_plans 投影状态为 rejected
2. 写入 EventStore（ExecutionPlanRejected 事件）
3. 通过 EventBus 路由到分解 Agent（触发重新分解）

## 7. Agent 管理 API

### 7.1 POST /projects/{id}/agents — 创建/注册 Agent

## 8. SSEChannel 事件流

### 8.1 GET /events/stream

SSEChannel 是 ChannelPlugin 的 MVP 实现（详见 1.2-mvp-event-infrastructure.md 第 8 章）。前端通过 SSE 长连接接收 Orbion 事件推送，Phase 2 IMChannel 将为 IM 平台提供相同的事件翻译和推送。

用户级 SSE 连接——每个已认证用户建立一条连接，接收其参与的所有项目的事件推送。无需指定 project_id，后端根据 JWT 中的用户身份自动筛选该用户参与的项目事件。

**认证：** JWT（query param: `?token={jwt}` 或 header）

**响应格式：** SSE 长连接

**推送事件类型：**

| SSE event | data 格式 |
|-----------|----------|
| project_created | `{"project_id":"proj-1","name":"project-name","description":"...","created_by":"user-1","created_at":"..."}` |
| message_created | `{"project_id":"proj-1","id":"msg-1","participant_id":"user-1","participant_type":"human","content":"hello","created_at":"..."}` |
| summary_generated | `{"project_id":"proj-1","id":"msg-2","participant_id":"agent-summary","participant_type":"agent","consensus_points":[...],"divergence_points":[...],"created_at":"..."}` |
| plan_proposed | `{"project_id":"proj-1","plan_id":"plan-1","thread_id":"thread-1","status":"proposed","tasks":[...]}` |
| plan_approved | `{"project_id":"proj-1","plan_id":"plan-1","status":"approved","approved_tasks":[...]}` |
| plan_rejected | `{"project_id":"proj-1","plan_id":"plan-1","status":"rejected","reason":"...","suggestions":[...]}` |
| output_generated | `{"project_id":"proj-1","output_id":"out-1","task_id":"t-1","plan_id":"plan-1","status":"generated","diff":"..."}` |
| output_approved | `{"project_id":"proj-1","output_id":"out-1","status":"approved","feedback":"..."}` |
| revision_requested | `{"project_id":"proj-1","output_id":"out-1","task_id":"t-1","issues":[...],"suggestions":[...]}` |
| agent_status_changed | `{"project_id":"proj-1","agent_id":"agent-1","status":"running","current_task":"t-1"}` |

前端监听这些 SSE 事件，根据 project_id 路由到对应项目视图，实时更新三栏内容。

## 10. FastAPI 路由组织

各模块的 Router 按职责分离：

```python
# app/main.py — lifespan初始化Provider
@asynccontextmanager
async def lifespan(app: FastAPI):
    settings = get_settings()
    # EventStore + EventBus 初始化（实现名从配置读取）
    store_cls = load_store_impl(settings.event_store)
    app.state.event_store = store_cls()
    await app.state.event_store.connect()
    app.state.event_bus = InProcessEventBus()
    # UserRepositoryProvider 初始化（实现名从配置读取）
    provider_cls = load_user_repo_provider(settings.user_repo)
    app.state.user_repo_provider = provider_cls()
    await app.state.user_repo_provider.connect()
    yield
    await app.state.event_store.close()
    await app.state.user_repo_provider.close()

app = FastAPI(title="Orbion MVP", lifespan=lifespan)

# 认证模块 — 不需要项目前缀
app.include_router(auth_routes.router, prefix="/auth", tags=["auth"])

app.include_router(plan_routes.router, prefix="/plans", tags=["plans"])

app.include_router(output_routes.router, prefix="/outputs", tags=["outputs"])
```
