## 1. JWT 认证流程

### 1.1 用户注册 POST /auth/register

**请求：**

```json
{
  "username": "string (3-32 chars, alphanumeric + underscore)",
  "password": "string (8+ chars)",
  "display_name": "string (1-64 chars)"
}
```

**响应 200：**

```json
{
  "user_id": "string (UUID)",
  "username": "string",
  "display_name": "string",
  "access_token": "string (JWT)",
  "token_type": "bearer"
}
```

**错误响应：**

- 400 — 用户名已存在 / 参数校验失败
- 格式：`{"detail": "Username already exists"}`

### 1.2 用户登录 POST /auth/login

**请求：**

```json
{
  "username": "string",
  "password": "string"
}
```

**响应 200：** 同 register

**错误响应：**

- 401 — 用户名或密码错误

### 1.3 JWT 结构

```json
{
  "sub": "user_id (UUID)",
  "username": "string",
  "exp": 1706000000,
  "iat": 1705990000
}
```

JWT payload 不包含角色信息。项目角色通过 `GET /projects` 响应中的 role 字段获取。这是设计规格的明确要求：项目角色信息不在 JWT 中，登录后通过 API 查询。

### 1.4 JWT 配置

| 参数 | 值 | 说明 |
|------|---|------|
| 算法 | HS256 | MVP 用对称密钥，简单高效 |
| 密钥 | 环境变量 JWT_SECRET | 不硬编码，从 config 读取 |
| 过期时间 | 7 天 | 自我构建阶段不需要频繁登录 |
| 签发者 | "orbion" | JWT iss 字段 |

### 1.5 FastAPI 认证依赖

```python
# auth/dependencies.py
async def get_current_user(token: str = Depends(oauth2_scheme)) -> User:
    """从 JWT token 解析当前用户。

    所有需要认证的端点注入此 dependency。
    返回 User 对象（user_id, username, display_name）。
    """
    payload = jwt.decode(token, config.jwt_secret, algorithms=["HS256"])
    user_id = payload["sub"]
    username = payload["username"]
    return User(id=user_id, username=username, display_name=payload.get("display_name", ""))
```

## 2. 项目管理 API

### 2.1 POST /projects — 创建项目

**认证：** 必须（get_current_user）

**请求：**

```json
{
  "name": "string (1-128 chars)",
  "description": "string | null"
}
```

**响应 200：**

```json
{
  "id": "string (UUID)",
  "name": "string",
  "description": "string | null",
  "tenant_id": "default",
  "created_at": "datetime"
}
```

**逻辑：** 创建者自动成为 Project Owner（roles = HumanPermission.all_bits()）

### 2.2 GET /projects — 列出用户参与的项目

**认证：** 必须

**响应 200：**

```json
[
  {
    "id": "string",
    "name": "string",
    "description": "string | null",
    "role": "owner | admin | member | viewer",
    "created_at": "datetime"
  }
]
```

**逻辑：** 只返回当前用户参与的项目。role 从 project_members 投影表计算。

### 2.3 GET /projects/{id} — 项目详情

**认证：** 必须，且用户必须是该项目成员

**响应 200：** 同上，单个项目对象

**错误响应：**

- 404 — 项目不存在 / 用户不是成员

### 2.4 POST /projects/{id}/members — 添加成员

**认证：** 必须，需要 Project Owner 或 Admin 权限

**请求：**

```json
{
  "user_id": "string (UUID)",
  "role": "owner | admin | member | viewer"
}
```

**响应 200：**

```json
{
  "participant_id": "string",
  "project_id": "string",
  "type": "human",
  "display_name": "string",
  "role": "string"
}
```

**错误响应：**

- 403 — 当前用户不是 Owner/Admin
- 404 — 目标用户不存在

## 3. 讨论线程 API

### 3.1 POST /projects/{id}/threads — 创建线程

**认证：** 必须，项目成员

**请求：**

```json
{
  "title": "string (1-256 chars)",
  "type": "discussion"
}
```

**响应 200：**

```json
{
  "id": "string (UUID)",
  "project_id": "string",
  "title": "string",
  "status": "active",
  "type": "discussion",
  "created_at": "datetime"
}
```

**逻辑：** 同时发布 DiscussionMessageCreated 事件（type="thread_created"），EventBus 路由到投影更新。

### 3.2 GET /projects/{id}/threads — 线程列表

**认证：** 必须，项目成员

**响应 200：**

```json
[
  {
    "id": "string",
    "title": "string",
    "status": "active | archived | resolved",
    "type": "discussion",
    "has_summary": true,
    "pending_plan_count": 2,
    "message_count": 5,
    "created_at": "datetime"
  }
]
```

**逻辑：**

- `has_summary` — 从 thread_messages 投影查询是否有 DiscussionSummaryGenerated 类型消息
- `pending_plan_count` — 从 execution_plans 投影查询 status="proposed" 且关联此线程的计划数
- `message_count` — 从 thread_messages 投影计算

## 4. 消息 API

### 4.1 POST /threads/{id}/messages — 发送消息

**认证：** 必须，项目成员

**请求：**

```json
{
  "content": "string (1-10000 chars)",
  "request_summary": false
}
```

**响应 200：**

```json
{
  "id": "string (UUID)",
  "thread_id": "string",
  "participant_id": "string",
  "participant_type": "human",
  "display_name": "string",
  "content": "string",
  "created_at": "datetime"
}
```

**逻辑：**

1. 写入 thread_messages 投影
2. 写入 EventStore（DiscussionMessageCreated 事件）
3. 通过 EventBus.publish 路由
4. 如果 `request_summary=true`，EventBus 传递到总结 Agent（payload 中附加此标志）

### 4.2 GET /threads/{id}/messages?before={msg_id}&limit=50 — 消息列表

**认证：** 必须，项目成员

**查询参数：**

- `before` — 游标分页，返回此 ID 之前的消息（逆序时间线）
- `limit` — 默认 50，最大 100

**响应 200：**

```json
[
  {
    "id": "string",
    "participant_id": "string",
    "participant_type": "human | agent",
    "display_name": "string",
    "content": "string",
    "event_type": "DiscussionMessageCreated | DiscussionSummaryGenerated | ExecutionPlanProposed",
    "created_at": "datetime"
  }
]
```

**逻辑：** 人类发言和 Agent 产出在同一条消息流中，通过 `participant_type` 和 `event_type` 区分样式。前端根据 `participant_type` 选择不同消息卡片样式（人类用普通样式，Agent 用特殊样式）。

## 5. 执行计划 API

### 5.1 GET /projects/{id}/plans — 计划列表

**认证：** 必须，项目成员

**查询参数：**

- `thread_id` — 可选，按线程过滤
- `status` — 可选，按状态过滤

**响应 200：**

```json
[
  {
    "id": "string",
    "thread_id": "string | null",
    "status": "proposed | approved | rejected | executing | completed",
    "proposed_by": "string (participant_id)",
    "tasks": [
      {
        "task_id": "string",
        "type": "code | document | configuration",
        "description": "string",
        "status": "pending | in_progress | completed",
        "dependencies": ["task_id"],
        "priority": "high | medium | low"
      }
    ],
    "created_at": "datetime"
  }
]
```

### 5.2 POST /plans/{id}/approve — 审批计划

**认证：** 必须，项目 Owner/Admin/Member（有 APPROVE_PLAN 权限位）

**请求：**

```json
{
  "approved_tasks": ["task_id1", "task_id2"],
  "modifications": {
    "task_id3": {
      "description": "修改后的描述"
    }
  }
}
```

**响应 200：**

```json
{
  "plan_id": "string",
  "status": "approved",
  "approved_tasks": ["task_id1", "task_id2"],
  "modifications": {}
}
```

**逻辑：**

1. 更新 execution_plans 投影状态
2. 写入 EventStore（ExecutionPlanApproved 事件）
3. 通过 EventBus 路由到执行 Agent
4. 未批准的任务不传递给执行 Agent

支持部分审批：只批准某些任务，对其他任务提供修改意见。

### 5.3 POST /plans/{id}/reject — 拒绝计划

**认证：** 同 approve

**请求：**

```json
{
  "reason": "string",
  "suggestions": ["string"]
}
```

**响应 200：**

```json
{
  "plan_id": "string",
  "status": "rejected",
  "reason": "string",
  "suggestions": ["string"]
}
```

**逻辑：**

1. 更新 execution_plans 投影状态为 rejected
2. 写入 EventStore（ExecutionPlanRejected 事件）
3. 通过 EventBus 路由到分解 Agent（触发重新分解）

## 6. 任务产出 API

### 6.1 GET /projects/{id}/outputs?plan_id={id} — 产出列表

**认证：** 必须，项目成员

**查询参数：**

- `plan_id` — 可选，按执行计划过滤

**响应 200：**

```json
[
  {
    "id": "string",
    "task_id": "string",
    "plan_id": "string",
    "output_type": "code | document",
    "content": "string",
    "diff": "string | null",
    "file_paths": ["string"],
    "status": "generated | approved | revision_requested",
    "version": 1,
    "created_at": "datetime"
  }
]
```

### 6.2 POST /outputs/{id}/approve — 审批产出

**认证：** 必须，项目成员（有 APPROVE_PLAN 权限位）

**请求：**

```json
{
  "feedback": "string | null"
}
```

**响应 200：**

```json
{
  "output_id": "string",
  "status": "approved",
  "feedback": "string | null"
}
```

**逻辑：**

1. 更新 task_outputs 投影状态为 approved
2. 写入 EventStore（TaskOutputApproved 事件）
3. **自动触发 Git commit** — git/service.py 将产出内容 commit 到本地仓库

### 6.3 POST /outputs/{id}/request-revision — 要求修改

**认证：** 同 approve

**请求：**

```json
{
  "issues": ["string"],
  "suggestions": ["string"]
}
```

**响应 200：**

```json
{
  "output_id": "string",
  "status": "revision_requested",
  "issues": ["string"],
  "suggestions": ["string"]
}
```

**逻辑：**

1. 更新 task_outputs 投影状态为 revision_requested
2. 写入 EventStore（TaskOutputRevisionRequested 事件）
3. 通过 EventBus 路由到执行 Agent — Agent 重新生成产出

## 7. Agent 管理 API

### 7.1 POST /projects/{id}/agents — 创建/注册 Agent

**认证：** 必须，Project Owner 或 Admin

**请求：**

```json
{
  "agent_type": "summary | decompose | execute",
  "model_id": "claude-haiku-4-5-20251001 | claude-sonnet-4-6",
  "display_name": "string"
}
```

**响应 200：**

```json
{
  "participant_id": "string",
  "project_id": "string",
  "type": "agent",
  "display_name": "string",
  "agent_type": "string",
  "model_id": "string",
  "status": "idle",
  "subscribed_events": ["DiscussionMessageCreated"],
  "roles": "bitmask value"
}
```

**逻辑：**

1. 在 project_members 投影中添加 Agent 成员
2. Agent Runtime 注册该 Agent（从预定义的 AgentDeclaration 模板初始化）
3. 根据 agent_type 自动分配 Agent 角色权限位

### 7.2 GET /projects/{id}/agents — Agent 列表

**认证：** 必须，项目成员

**响应 200：** 同上，列表形式

### 7.3 GET /projects/{id}/agents/{id}/status — Agent 状态

**认证：** 必须，项目成员

**响应 200：**

```json
{
  "agent_id": "string",
  "status": "idle | running | error",  // paused 为 Phase 2 预留，MVP 不使用
  "current_task": "string | null",
  "completed_count": 12,
  "error_count": 0,
  "last_execution_at": "datetime | null"
}
```

## 8. SSEChannel 事件流

### 8.1 GET /events/stream?project_id={id}

SSEChannel 是 ChannelPlugin 的 MVP 实现（详见 1.2-mvp-event-infrastructure.md 第 8 章）。前端通过 SSE 长连接接收 Orbion 事件推送，Phase 2 IMChannel 将为 IM 平台提供相同的事件翻译和推送。

**认证：** JWT（query param: `?token={jwt}` 或 header）

**响应格式：** SSE 长连接

**推送事件类型：**

| SSE event | data 格式 |
|-----------|----------|
| message_created | `{"id":"msg-1","participant_id":"user-1","participant_type":"human","content":"hello","created_at":"..."}` |
| summary_generated | `{"id":"msg-2","participant_id":"agent-summary","participant_type":"agent","consensus_points":[...],"divergence_points":[...],"created_at":"..."}` |
| plan_proposed | `{"plan_id":"plan-1","thread_id":"thread-1","status":"proposed","tasks":[...]}` |
| plan_approved | `{"plan_id":"plan-1","status":"approved","approved_tasks":[...]}` |
| plan_rejected | `{"plan_id":"plan-1","status":"rejected","reason":"...","suggestions":[...]}` |
| output_generated | `{"output_id":"out-1","task_id":"t-1","plan_id":"plan-1","status":"generated","diff":"..."}` |
| output_approved | `{"output_id":"out-1","status":"approved","feedback":"..."}` |
| revision_requested | `{"output_id":"out-1","task_id":"t-1","issues":[...],"suggestions":[...]}` |
| agent_status_changed | `{"agent_id":"agent-1","status":"running","current_task":"t-1"}` |

前端监听这些 SSE 事件，实时更新三栏内容。

## 9. 错误响应格式

所有错误响应统一格式：

```json
{
  "detail": "string"
}
```

| HTTP 状态码 | 使用场景 |
|------------|---------|
| 400 | 参数校验失败（Pydantic 自动处理） |
| 401 | 未认证或 JWT 过期 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 409 | 冲突（如用户名已存在） |
| 500 | 服务内部错误 |

## 10. FastAPI 路由组织

各模块的 Router 按职责分离：

```python
# app/main.py
app = FastAPI(title="Orbion MVP")

# 认证模块 — 不需要项目前缀
app.include_router(auth_routes.router, prefix="/auth", tags=["auth"])

# 项目模块 — /projects 前缀
app.include_router(project_routes.router, prefix="/projects", tags=["projects"])

# 线程模块 — /projects/{id}/threads 前缀
app.include_router(thread_routes.router, prefix="/projects/{project_id}/threads", tags=["threads"])

# 消息模块 — /threads/{id}/messages 前缀
app.include_router(message_routes.router, prefix="/threads/{thread_id}/messages", tags=["messages"])

# 计划模块 — /plans 前缀
app.include_router(plan_routes.router, prefix="/plans", tags=["plans"])

# 产出模块 — /outputs 前缀
app.include_router(output_routes.router, prefix="/outputs", tags=["outputs"])

# Agent 模块 — /projects/{id}/agents 前缀
app.include_router(agent_routes.router, prefix="/projects/{project_id}/agents", tags=["agents"])

# SSE 流 — /events 前缀
app.include_router(sse_routes.router, prefix="/events", tags=["events"])
```

| 消息 | /threads/{id}/messages | GET | 必须 | 项目成员 |
| 计划 | /projects/{id}/plans | GET | 必须 | 项目成员 |
| 计划 | /plans/{id}/approve | POST | 必须 | APPROVE_PLAN 权限位 |
| 计划 | /plans/{id}/reject | POST | 必须 | REJECT_PLAN 权限位 |
| 产出 | /projects/{id}/outputs | GET | 必须 | 项目成员 |
| 产出 | /outputs/{id}/approve | POST | 必须 | APPROVE_PLAN 权限位 |
| 产出 | /outputs/{id}/request-revision | POST | 必须 | 项目成员 |
| Agent | /projects/{id}/agents | POST | 必须 | Owner/Admin |
| Agent | /projects/{id}/agents | GET | 必须 | 项目成员 |
| Agent | /projects/{id}/agents/{id}/status | GET | 必须 | 项目成员 |
| SSEChannel | /events/stream | GET | JWT | 项目成员 |

共 **18 个端点 + 1 个 SSE 流**。比原始 API 契约多了 GET /projects/{id}/members 和 GET /projects/{id}/plans 的查询参数扩展。

## 12. MVP 范围裁剪

| 功能 | MVP | Phase 2 |
|------|-----|---------|
| 认证 | 用户名密码 + JWT | + OAuth (GitHub/Google) |
| Token 刷新 | 不做（7 天过期足够） | Refresh token 机制 |
| 用户管理 | 仅 register/login | 用户详情编辑、头像 |
| 项目设置 | 不做 | 项目配置 API |
| 线程归档/恢复 | 不做 | 线程状态变更 API |
