## 1. PostgreSQL Schema

### 1.1 event_log — 不可变事件日志

写端核心表，所有事件以不可变日志存储。

```sql
CREATE TABLE event_log (
    event_id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id       VARCHAR(64) NOT NULL,
    event_type       VARCHAR(64) NOT NULL,
    participant_id   VARCHAR(64) NOT NULL,
    participant_type VARCHAR(8)  NOT NULL CHECK (participant_type IN ('human', 'agent')),
    payload          JSONB       NOT NULL DEFAULT '{}',
    correlation_id   UUID        NOT NULL,
    causation_id     UUID        NULL,
    created_at       TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 项目边界硬隔离查询（最频繁的查询模式）
CREATE INDEX idx_event_log_project    ON event_log (project_id, created_at DESC);
-- correlation_id 链路追踪（Agent 上下文组装）
CREATE INDEX idx_event_log_correlation ON event_log (correlation_id, created_at);
-- 事件类型过滤（Agent 订阅特定事件类型时查询）
CREATE INDEX idx_event_log_type       ON event_log (project_id, event_type);
```

**设计要点：**

- event_id 使用 gen_random_uuid() 自动生成，不需要应用层传入
- 所有查询必须带 project_id（项目边界硬隔离）
- payload 是 JSONB，不同事件类型的 payload 结构不同（见 mvp-event-infrastructure.md）
- causation_id 为 NULL 表示人类直接发起的事件

### 1.4 project_members — 项目成员投影表

读端投影表，人类和 Agent 统一存储。

```sql
CREATE TABLE project_members (
    participant_id  VARCHAR(64) NOT NULL,  -- 人类: user_id, Agent: agent_id
    project_id      UUID        NOT NULL REFERENCES projects(id),
    type            VARCHAR(8)  NOT NULL CHECK (type IN ('human', 'agent')),
    display_name    VARCHAR(64) NOT NULL,
    roles           BIGINT      NOT NULL DEFAULT 0,  -- bitmask 权限位
    -- Agent 专属字段（type='agent' 时有值）
    agent_type      VARCHAR(32) NULL,       -- summary/decompose/execute
    model_id        VARCHAR(64) NULL,       -- claude-haiku-4-5-20251001 等
    status          VARCHAR(16) NULL DEFAULT 'idle',  -- idle/running/error（paused 为 Phase 2 预留，MVP 不使用）
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (participant_id, project_id)
);

CREATE INDEX idx_project_members_project ON project_members (project_id, type);
CREATE INDEX idx_project_members_agent   ON project_members (project_id, agent_type) WHERE type = 'agent';
```

**设计要点：**

- 人类和 Agent 在同一张表，通过 `type` 区分
- `roles` 是 bitmask（见 mvp-permissions.md）
- Agent 专属字段在 type='agent' 时有值，人类时为 NULL
- PRIMARY KEY 是 (participant_id, project_id)，同一用户可参与多个项目

### 1.5 threads — 讨论线程表

```sql
CREATE TABLE threads (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID        NOT NULL REFERENCES projects(id),
    title           VARCHAR(256) NOT NULL,
    status          VARCHAR(16) NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'archived', 'resolved')),
    type            VARCHAR(32) NOT NULL DEFAULT 'discussion',
    created_by      VARCHAR(64) NOT NULL,  -- participant_id
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_threads_project ON threads (project_id, created_at DESC);
```

### 1.6 thread_messages — 线程消息投影表

读端投影表，人类发言和 Agent 产出在同一条消息流中。

```sql
CREATE TABLE thread_messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    thread_id       UUID        NOT NULL REFERENCES threads(id),
    project_id      UUID        NOT NULL,
    participant_id  VARCHAR(64) NOT NULL,
    participant_type VARCHAR(8) NOT NULL CHECK (participant_type IN ('human', 'agent')),
    display_name    VARCHAR(64) NOT NULL,
    content         TEXT        NOT NULL,
    event_type      VARCHAR(64) NOT NULL,  -- DiscussionMessageCreated/DiscussionSummaryGenerated/...
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_thread_messages_thread ON thread_messages (thread_id, created_at);
CREATE INDEX idx_thread_messages_summary ON thread_messages (thread_id, event_type)
    WHERE event_type = 'DiscussionSummaryGenerated';
```

**设计要点：**

- `event_type` 区分消息来源类型，前端根据此字段选择不同卡片样式
- 部分索引 `idx_thread_messages_summary` 用于快速查询线程是否有摘要（has_summary 字段）

### 1.7 execution_plans — 执行计划投影表

读端投影表。

```sql
CREATE TABLE execution_plans (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID        NOT NULL,
    thread_id       UUID        NULL REFERENCES threads(id),  -- 与讨论线程关联
    correlation_id  UUID        NOT NULL,  -- 链路追踪 ID
    status          VARCHAR(16) NOT NULL DEFAULT 'proposed'
        CHECK (status IN ('proposed', 'approved', 'rejected', 'executing', 'completed')),
    proposed_by     VARCHAR(64) NOT NULL,  -- agent participant_id
    approved_by     JSONB       NOT NULL DEFAULT '[]',  -- [{"participant_id": "...", "approved_at": "..."}]
    tasks           JSONB       NOT NULL DEFAULT '[]',  -- 任务列表
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_execution_plans_project  ON execution_plans (project_id, created_at DESC);
CREATE INDEX idx_execution_plans_thread   ON execution_plans (thread_id) WHERE thread_id IS NOT NULL;
CREATE INDEX idx_execution_plans_status   ON execution_plans (project_id, status) WHERE status = 'proposed';
```

**tasks JSONB 结构：**

```json
[
  {
    "task_id": "string",
    "type": "code | document | configuration",
    "description": "string",
    "dependencies": ["task_id"],
    "priority": "high | medium | low",
    "status": "pending | in_progress | completed"
  }
]
```

**部分索引 `idx_execution_plans_status`** 用于快速查询 pending 计划数（线程列表的 pending_plan_count 字段）。

### 1.8 task_outputs — 任务产出投影表

读端投影表。

```sql
CREATE TABLE task_outputs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id      UUID        NOT NULL,
    task_id         VARCHAR(64) NOT NULL,
    plan_id         UUID        NOT NULL REFERENCES execution_plans(id),
    output_type     VARCHAR(16) NOT NULL CHECK (output_type IN ('code', 'document')),
    content         TEXT        NOT NULL,
    diff            TEXT        NULL,       -- 代码 diff（output_type='code' 时）
    file_paths      JSONB       NOT NULL DEFAULT '[]',  -- ["path/to/file.py"]
    status          VARCHAR(24) NOT NULL DEFAULT 'generated'
        CHECK (status IN ('generated', 'approved', 'revision_requested')),
    version         INT         NOT NULL DEFAULT 1,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_task_outputs_project ON task_outputs (project_id, plan_id);
CREATE INDEX idx_task_outputs_status  ON task_outputs (project_id, status);
```

## 4. Event Store 与投影表的读写分离

### 4.1 写端

> CQRS Command Side

- **唯一写入口**：EventStore.append()
- 所有业务操作先写入 event_log，再通过 EventBus 触发投影更新
- event_log 是不可变日志，永不修改或删除

### 4.2 读端

> CQRS Query Side

- **前端查询**全部走投影表（thread_messages、execution_plans 等）
- 投影表由 EventBus handler 异步更新，不阻塞写端
- 投影表数据可从 event_log 完全重建：如果投影表数据异常，删掉重建即可

### 4.3 读写流程

```
业务操作 → service.py
  1. 写入 event_log（EventStore.append）
  2. 发布到 EventBus（EventBus.publish）
  3. EventBus handler 更新投影表（EventProjections）
  4. SSEChannel 推送给前端
```

前端请求 → service.py → 读取投影表 → 返回响应

### 4.4 信息分级存储策略

> 架构规格映射：第 4 章「Event Store & CQRS」第 4.4 节

架构规格定义 4 级信息分级（详见 0.1-arch.md 第 4.4 节和 1.2-mvp-event-infrastructure.md 第 4.4 节）。数据模型层面的 MVP 简化：

| 级别 | MVP 数据模型影响 |
|------|---------------|
| Level 0（全量永久） | 人类输入事件 payload 全量存 event_log.payload JSONB |
| Level 1（Agent 产出） | MVP 简化：payload 全量存 event_log.payload JSONB，不做大 payload 外置 |
| Level 2（Agent 过程） | MVP 不做 TTL 存储，Agent 执行过程不进 event_log |
| Level 3（系统操作） | SSE 连接管理等只进 Python logging，不建表 |

MVP event_log 表所有事件 payload 统一存 JSONB，不区分 Level 0 和 Level 1。Phase 2 引入大 payload 外置时，event_log.payload 字段将改为存储引用 ID，大 payload 存对象存储。

## 5. 项目边界硬隔离

所有涉及项目数据的查询必须带 project_id，确保项目间数据不可交叉访问。

### 5.1 实现方式

1. **event_log 索引**：所有索引都以 project_id 为前缀列
2. **投影表**：所有投影表包含 project_id 字段
3. **API 端点**：所有查询从 URL path 提取 project_id，不依赖前端传入
4. **权限检查**：先验证用户是该项目成员，再执行查询

```python
# 伪代码：所有 service 方法的第一步
async def verify_project_membership(user_id: str, project_id: str) -> None:
    member = await get_member(user_id, project_id)
    if not member:
        raise HTTPException(404, "Project not found or not a member")
```

## 6. MVP 范围裁剪

| 功能 | MVP | Phase 2 |
|------|-----|---------|
| 数据库 | PostgreSQL only | 不变 |
| ORM | asyncpg 直接 SQL | 可引入 SQLAlchemy |
| 迁移 | 手动 SQL 文件 | alembic 管理 |
| 投影重建 | 不做 | EventStore → 重建投影 |
| 信息分级 | payload 统一存 JSONB，不区分 Level 0/1 | Level 1 大 payload 外置 + Level 2 TTL 存储 |
| 消息编辑/删除 | 不做 | 消息版本管理 |
| 线程归档 | 不做（只建不归档） | 线程状态变更 |
| 知识库表 | 不做 | documents + link_graph 表 |
