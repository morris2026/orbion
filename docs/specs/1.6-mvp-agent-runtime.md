## 1. MVP 三个 Agent 定义

MVP 包含 3 个内置 Agent，不可自定义。Phase 2 才支持自定义 Agent。

### 1.1 总结 Agent

```python
summary_declaration = AgentDeclaration(
    agent_id="agent-summary-template",  # 模板 ID，实际注册时生成唯一 ID
    agent_type="summary",              # Agent 类型标识，用于调度路由和权限分配
    display_name="总结助手",
    role="讨论总结专家",
    goal="从讨论线程中提炼共识点、分歧点和行动项",
    backstory=(
        "你是一个讨论总结 Agent。你订阅讨论线程中的消息事件，"
        "当收到 request_summary 标志或讨论消息累积到阈值时，"
        "你从 EventStore 获取该线程的完整消息历史，"
        "提炼共识点（所有人都同意的结论）、分歧点（尚有争议的观点）、"
        "行动项（需要后续执行的任务）。"
        "输出格式必须是结构化 JSON。"
    ),
    subscribed_events=["DiscussionMessageCreated"],
    output_event_type="DiscussionSummaryGenerated",
    model_config=ModelConfig(
        model_id="claude-haiku-4-5-20251001",
        temperature=0.3,   # 低温度，保证总结结构化
        max_tokens=2048,
    ),
    capabilities=["discussion.summarize", "knowledge.query"],
    skills=[
        SkillDeclaration(
            name="discussion.summarize",
            description="总结讨论线程内容",
            input_schema={"thread_id": "string"},
            output_schema={"consensus_points": "list", "divergence_points": "list", "action_items": "list"},
            risk_level="low",
            requires_approval=False,
        ),
    ],
    max_concurrent_tasks=1,
    requires_approval=False,   # 总结产出不需要人类审批（低风险）
    checkpoint_on_output=False,
)
```

**触发条件：**

- 收到 DiscussionMessageCreated 事件，payload 中 request_summary=true
- 或者：同一线程消息数达到阈值（默认 10 条，可配置）

**触发逻辑：**

- request_summary=true 时立即触发
- 阈值触发时，scheduler 检查线程消息数是否达到阈值

## 2. Agent 生命周期状态机

### 2.1 状态定义

```
IDLE → RUNNING → PAUSED → RESUMED → RUNNING → ...
                 ↓              ↓
               OUTPUT          OUTPUT
                 ↓              ↓
            (awaiting      (awaiting
             approval)     approval)
                                 ↓
                              COMPLETED

IDLE → RUNNING → ERROR → IDLE (after retry/fix)
```

### 2.2 状态转换规则

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|---------|---------|------|
| IDLE | 收到订阅事件 | RUNNING | 开始处理任务 |
| RUNNING | 产出事件完成 + requires_approval=True | PAUSED | 等待人类审批 |
| RUNNING | 产出事件完成 + requires_approval=False | IDLE | 低风险产出，不需要审批 |
| PAUSED | 收到审批通过事件 | RESUMED | 审批恢复 |
| RESUMED | 自动继续执行 | RUNNING → IDLE | 审批后继续或完成 |
| RUNNING | 执行失败 | ERROR | 记录错误信息 |
| ERROR | 错误已修复/重试 | IDLE | 重试后回到空闲 |

### 2.3 MVP 简化

MVP 不实现完整的 checkpoint 暂停/恢复机制。简化为：

- Agent 产出后直接发布事件到 EventBus
- `requires_approval=True` 的 Agent 不暂停，只是产出状态为 "generated" 等待人类审批
- 人类审批/拒绝后，审批事件通过 EventBus 路由回 Agent
- Agent 不需要"暂停"状态 — 它只是产出后回到 IDLE，下次收到相关事件时再处理

这意味着 MVP 的 Agent 状态只有：**idle / running / error**

PAUSED/RESUMED 状态在 Phase 2 实现完整的 LangGraph checkpoint 机制时才引入。

from typing import Protocol

class ModelAdapter(Protocol):

        """调用 LLM 完成任务。

        Args:
            prompt: 组装好的 prompt 输入

        Returns:
            ModelOutput: LLM 产出结果
        """
        ...

### 3.2 PromptInput 结构

```python
class PromptInput(BaseModel):
    """Agent prompt 组装的统一输入格式。"""
    system_prompt: str              # role + goal + backstory → 系统提示
    context: str                    # KnowledgeRetriever 注入的知识上下文（MVP 为空）
    memory: str                     # memory.md 加载的行为偏好
    task: str                       # 事件 payload 转化的任务描述
    history: list[EventSummary]     # 同 correlation_id 的历史事件摘要

class EventSummary(BaseModel):
    """事件摘要，用于构建 Agent 上下文历史。"""
    event_type: str
    participant_id: str
    participant_type: str    # human/agent — 用于角色映射（user/assistant）
    content: str     # 事件关键内容摘要（不是完整 payload）
    created_at: datetime
```

### 3.3 ModelOutput 结构

```python
class ModelOutput(BaseModel):
    """LLM 产出的统一输出格式。"""
    content: str                    # 产出内容（结构化 JSON 或纯文本）
    skill_calls: list[SkillCall] | None = None  # Agent 调用的 Skill 列表
    reasoning: str | None = None   # Agent 的推理过程（可选）

class SkillCall(BaseModel):
    """Agent 对 Skill 的调用请求。"""
    name: str                       # Skill 名称
    arguments: dict[str, Any]       # Skill 参数
```

class ModelConfig(BaseModel):

## 4. ClaudeAdapter — MVP 实现

```python
class ClaudeAdapter:
    """Anthropic Claude SDK 适配器，MVP 使用。"""

    def __init__(self, api_key: str):
        self._client = anthropic.AsyncAnthropic(api_key=api_key)

    async def complete(self, prompt: PromptInput) -> ModelOutput:
        """调用 Claude API 生成产出。"""
        # 组装 Claude API 请求格式
        messages = self._build_messages(prompt)
        system = self._build_system(prompt)

        response = await self._client.messages.create(
            model=prompt.model_config.model_id,  # 从 runtime 传入
            max_tokens=prompt.model_config.max_tokens,
            temperature=prompt.model_config.temperature,
            system=system,
            messages=messages,
        )

        # 解析产出
        content = response.content[0].text
        return self._parse_output(content)

    def _build_system(self, prompt: PromptInput) -> str:
        """组装系统提示：role + goal + backstory + memory + context"""
        parts = [
            prompt.system_prompt,   # Agent role/goal/backstory
        ]
        if prompt.memory:
            parts.append(f"\n## 行为偏好\n{prompt.memory}")
        if prompt.context:
            parts.append(f"\n## 知识上下文\n{prompt.context}")
        return "\n\n".join(parts)

    def _build_messages(self, prompt: PromptInput) -> list[dict]:
        """组装对话消息：history + task"""
        messages = []
        for summary in prompt.history:
            messages.append({
                "role": "user" if summary.participant_type == "human" else "assistant",
                "content": summary.content,
            })
        messages.append({
            "role": "user",
            "content": prompt.task,
        })
        return messages

    def _parse_output(self, content: str) -> ModelOutput:
        """解析 LLM 产出为 ModelOutput。

        尝试解析 JSON 结构化产出。如果 LLM 输出的是 JSON，
        提取 skill_calls 和 reasoning。如果是纯文本，
        直接作为 content。
        """
        # MVP 简化：不做复杂的 JSON 解析
        # 要求 Agent backstory 中明确输出格式要求
        # LLM 产出格式由 Agent backstory 约定，runtime 层不做强制解析
        return ModelOutput(content=content)
```

## 5. Prompt 组装流程

Agent 收到事件后的完整 prompt 组装流程（7 步）：

```
1. AgentDeclaration → system_prompt
   从 Agent 的 role + goal + backstory 组装系统提示

2. EventStore.get_events_by_correlation → history
   用 correlation_id 获取整个协作链的历史事件，转化为 EventSummary 列表

3. KnowledgeRetriever.retrieve → context
   MVP 不实现，context 为空字符串。
   Phase 2 通过标签/链接检索相关知识文档注入

4. AgentMemory.load_memory_chain → memory
   按层次加载 memory.md：平台 → 项目 → Agent → 任务

5. 事件 payload → task
   将当前事件 payload 转化为自然语言任务描述

6. 合并为 PromptInput → ModelAdapter.complete()
   system_prompt + context + memory + history + task → LLM 调用

7. 解析 ModelOutput → 发布新事件
   解析 LLM 产出，构建新事件的 payload，发布到 EventBus
```

### 5.1 各步的输入输出

| 步骤 | 输入 | 输出 |
|------|------|------|
| 1 | AgentDeclaration.role/goal/backstory | system_prompt (str) |
| 2 | correlation_id | history (list[EventSummary]) |
| 3 | KnowledgeQuery | context (str) — MVP 为空 |
| 4 | project_id, agent_type | memory (str) |
| 5 | event payload | task (str) |
| 6 | PromptInput | ModelOutput |
| 7 | ModelOutput | 新事件 payload → EventBus.publish |

### 5.2 task 描述转换规则

不同事件类型的 payload 转化为自然语言任务描述：

| 事件类型 | 转换规则 |
|----------|---------|
| DiscussionMessageCreated | "请总结以下讨论线程的消息内容：{content}" |
| DiscussionSummaryGenerated | "请根据以下共识点和行动项，分解为可执行的代码任务：{consensus_points}, {action_items}" |
| ExecutionPlanApproved | "请执行以下审批通过的代码任务：{approved_tasks}" |
| TaskOutputRevisionRequested | "请根据以下修改意见重新生成产出：{issues}, {suggestions}" |

## 6. AgentScheduler — 事件调度器

AgentScheduler 负责接收 EventBus 事件，路由到对应 Agent。

```python
class AgentScheduler:
    """事件 → Agent 调度器。

    订阅 EventBus 的所有 MVP 事件类型，
    根据事件类型路由到对应 Agent 的 runtime.dispatch()。
    """

    def __init__(self, event_bus: EventBus, runtime: AgentRuntime):
        self._bus = event_bus
        self._runtime = runtime
        # 注册事件订阅
        self._bus.subscribe("DiscussionMessageCreated", self._on_message_created)
        self._bus.subscribe("DiscussionSummaryGenerated", self._on_summary_generated)
        self._bus.subscribe("ExecutionPlanApproved", self._on_plan_approved)
        self._bus.subscribe("TaskOutputRevisionRequested", self._on_revision_requested)

    async def _on_message_created(self, payload: dict) -> None:
        """讨论消息 → 检查是否触发总结 Agent。"""
        if payload.get("request_summary", False):
            # 立即触发总结 Agent
            await self._runtime.dispatch("summary", payload)
        # 阈值触发逻辑：检查线程消息数
        # MVP 简化：只支持 request_summary 标志触发

    async def _on_summary_generated(self, payload: dict) -> None:
        """讨论摘要 → 触发分解 Agent。"""
        await self._runtime.dispatch("decompose", payload)

    async def _on_plan_approved(self, payload: dict) -> None:
        """审批通过 → 触发执行 Agent。"""
        await self._runtime.dispatch("execute", payload)

    async def _on_revision_requested(self, payload: dict) -> None:
        """修改要求 → 触发执行 Agent 重新生成。"""
        await self._runtime.dispatch("execute", payload)
```

## 7. AgentRuntime — 生命周期管理

```python
class AgentRuntime:
    """Agent 生命周期管理和调度执行。"""

    def __init__(
        self,
        event_bus: EventBus,
        event_store: EventStore,
        projections: EventProjections,
        memory: AgentMemory,
        model_adapter: ModelAdapter,  # MVP: ClaudeAdapter
    ):
        self._bus = event_bus
        self._store = event_store
        self._projections = projections
        self._memory = memory
        self._adapter = model_adapter
        self._agents: dict[str, AgentDeclaration] = {}  # agent_type → declaration

    def register(self, declaration: AgentDeclaration) -> None:
        """注册 Agent 声明。"""
        self._agents[declaration.agent_type] = declaration

    async def dispatch(self, agent_type: str, event_payload: dict) -> None:
        """调度 Agent 处理事件。

        MVP 实现是同步执行（不并发）：
        1. 组装 PromptInput
        2. 调用 ModelAdapter.complete
        3. 解析产出
        4. 发布新事件到 EventBus
        """
        declaration = self._agents[agent_type]

        # 1. 组装 prompt
        prompt = await self._assemble_prompt(declaration, event_payload)

        # 2. 调用 LLM
        output = await self._adapter.complete(prompt)

        # 3. 解析产出，构建新事件 payload
        new_payload = self._parse_output_to_payload(agent_type, output, event_payload)

        # 4. 持久化 + 路由
        correlation_id = event_payload.get("correlation_id", "")
        event = Event(
            event_id=str(uuid4()),
            project_id=event_payload.get("project_id", ""),
            event_type=declaration.output_event_type,
            participant_id=declaration.agent_id,
            participant_type="agent",
            payload=new_payload,
            correlation_id=correlation_id,
            causation_id=event_payload.get("event_id"),
            created_at=datetime.now(timezone.utc),
        )
        await self._store.append(event)
        await self._bus.publish(declaration.output_event_type, new_payload)

    async def _assemble_prompt(
        self, declaration: AgentDeclaration, event_payload: dict
    ) -> PromptInput:
        """7 步 prompt 组装流程。"""
        # Step 1: system_prompt from declaration
        system_prompt = f"Role: {declaration.role}\nGoal: {declaration.goal}\n\n{declaration.backstory}"

        # Step 2: history from EventStore
        correlation_id = event_payload.get("correlation_id", "")
        events = await self._store.get_events_by_correlation(correlation_id)
        history = [
            EventSummary(
                event_type=e.event_type,
                participant_id=e.participant_id,
                participant_type=e.participant_type,
                content=str(e.payload)[:200],  # MVP 简化：截取前200字符作为摘要
                created_at=e.created_at,
            ) for e in events
        ]

        # Step 3: context — MVP 为空
        context = ""

        # Step 4: memory from AgentMemory
        project_id = event_payload.get("project_id", "")
        memory = self._memory.load_memory_chain(project_id, declaration.agent_type)

        # Step 5: task from payload
        task = self._payload_to_task(declaration.agent_type, event_payload)

        return PromptInput(
            system_prompt=system_prompt,
            context=context,
            memory=memory,
            task=task,
            history=history,
        )
```

## 8. memory.md 管理设计

### 8.1 层次结构

所有层级统一用 `memory.md` 文件名，路径表达作用范围：

```
memory.md                                   → 平台级（所有项目、所有 Agent）
project/{id}/memory.md                      → 项目级（该项目所有 Agent）
project/{id}/agents/{agent_type}/memory.md  → Agent级（特定 Agent 类型）
```

任务级记忆不做——链路上下文由 Event Store 按 correlation_id 提供，memory.md 只解决跨 correlation_id 的持久偏好。

### 8.2 AgentMemory 接口

```python
class AgentMemory:
    """Agent 层次化记忆管理。"""

    def __init__(self, base_path: str):
        """base_path 是 memory.md 文件存储根目录。
        MVP 存储在本地文件系统，Phase 2 可迁移到 PostgreSQL 或对象存储。
        """
        self._base = Path(base_path)

    def load_memory_chain(
        self, project_id: str, agent_type: str
    ) -> str:
        """按层次加载 memory.md：平台 → 项目 → Agent。

        后加载覆盖前面的设置（类似 CSS 层叠）。
        """
        ...

    def read_memory(self, path: str) -> str:
        """读取指定路径的 memory.md。"""
        ...

    def write_memory(self, path: str, content: str) -> None:
        """写入指定路径的 memory.md。"""
        ...

    def reset_agent_memory(self, project_id: str, agent_type: str) -> None:
        """重置特定 Agent 的记忆（清空内容，不删除文件）。"""
        ...
```

## 9. MVP 范围裁剪

| 功能 | MVP | Phase 2 |
|------|-----|---------|
| Agent 种类 | 3 个内置（总结/分解/执行） | 自定义 Agent 注册 |
| Agent 状态 | idle/running/error | + paused/resumed（checkpoint 机制） |
| 模型适配 | ClaudeAdapter only | + OpenAIAdapter、GeminiAdapter |
| Prompt 组装 | 7 步流程（context 步为空） | + KnowledgeRetriever 知识注入 |
| 记忆存储 | 本地文件系统 | PostgreSQL 或对象存储 |
| Skill 调用 | 不解析 skill_calls | 结构化 Skill 调用解析和执行 |
| Saga 补偿 | 不做 | Agent 失败自动补偿 |
| Agent 并发 | 不并发（串行执行） | max_concurrent_tasks 控制 |
| Agent 委派 | 不做 | 子 Agent 委派机制 |
| 记忆自动更新 | 不做 | Agent 产出附带 experience_update |
