## 1. ChannelAdapter Protocol 详细定义

```python
class ChannelAdapter(Protocol):
    """Channel 适配器抽象接口（Hub Gateway 内所有 Channel 共用）。

    SSEChannel 和 IMChannel 都实现此接口，与 ModelAdapter 同级：
    ModelAdapter 翻译 LLM 协议，ChannelAdapter 翻译外部通道协议。
    业务代码只依赖此接口，不关心消息来自 SSE 还是 IM。
    """

    async def receive(self, channel_event: ChannelEvent) -> None:
        """接收 IM 事件，翻译为 Orbion 事件，发布到 Event Bus。

        流程：
        1. 解析 IM 平台原始事件（Slack Event / WeChat XML / Discord Gateway Event）
        2. 翻译为 ChannelEvent 中间格式
        3. 映射到 Orbion 事件类型（DiscussionMessageCreated 等）
        4. 通过 Hub Gateway 内的 Event Bus 发布
        """
        ...

    async def send(self, orbion_event_type: str, payload: dict) -> None:
        """将 Orbion 事件翻译为 IM 消息，推送到 IM 平台。

        流程：
        1. 识别事件类型和目标项目
        2. 查询该项目的通道配置（channel_connections 表）
        3. 将 Orbion 事件 payload 翻译为目标通道的消息格式
        4. 通过目标通道 API 推送
        """
        ...

    async def send_approval(self, plan_payload: dict) -> None:
        """将执行计划翻译为 IM 审批交互。

        不同 IM 平台的审批交互形式不同：
        - Slack: Block Kit Interactive Buttons
        - Discord: Reaction emoji（👍/👎）
        - 企业微信/钉钉/飞书: 交互卡片按钮
        - Telegram: Inline Keyboard 按钮
        """
        ...

    def get_channel_type(self) -> str:
        """返回通道类型标识，如 "slack"/"wechat"/"discord"。"""
        ...
```

## 2. ChannelEvent 详细定义

```python
class ChannelEvent(BaseModel):
    """Channel 事件的标准化中间格式（所有 Channel 共用）。

    SSE 和 IM 的原始事件都翻译为此格式后，
    再映射到 Orbion 事件类型。类似 ModelAdapter 的 PromptInput
    是 LLM 调用的中间格式，ChannelEvent 是通道翻译的中间格式。
    """
    channel_type: str          # slack/wechat/discord/telegram/dingtalk/feishu
    channel_id: str            # IM 平台的 channel/group ID（映射到 Orbion project_id）
    sender_id: str             # IM 平台用户 ID（通过 channel_user_mapping 映射到 Orbion user_id）
    sender_name: str           # IM 平台显示名
    thread_id: str | None      # IM 平台的 thread/conversation ID（映射到 Orbion thread_id）
    content: str               # 消息文本内容
    action: str | None         # 用户交互动作：approve/reject/reply/summary_request
    action_payload: dict | None # 动作附加数据（审批的 task_id、拒绝的 reason 等）
    attachments: list[str]     # 文件/图片 URL
```

## 3. 双向消息流

### 3.1 入站流

```
┌─────────────────────────────────────────────────────────────────┐
│                     入站流（IM → Orbion）                        │
│                                                                 │
│  IM 用户发消息      IM 平台 Webhook       ChannelAdapter         │
│  ──────────→   ──────────────→   ──────────────→                │
│                      │                 │                         │
│  Slack Event     Slack API        receive()                      │
│  WeChat XML      WeChat API       → ChannelEvent 中间格式        │
│  Discord GW      Discord API      → 映射到 Orbion 事件类型      │
│                                    → Event Bus.publish()         │
│                                                                 │
│  映射规则：                                                     │
│  ChannelEvent.action=None  → DiscussionMessageCreated            │
│  ChannelEvent.action=approve → ExecutionPlanApproved             │
│  ChannelEvent.action=reject  → ExecutionPlanRejected             │
│  ChannelEvent.action=summary_request → payload.request_summary=true │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                     出站流（Orbion → IM）                        │
│                                                                 │
│  Agent 产出事件     Event Bus 路由        ChannelAdapter         │
│  ──────────→   ──────────────→   ──────────────→                │
│                      │                 │                         │
│  SummaryGenerated  subscribe()       send()                     │
│  PlanProposed      → payload        → IM 平台消息格式           │
│  OutputGenerated   → 查询项目通道   → IM 平台 API 推送          │
│                                    → Slack Block Kit /           │
│                                      企业微信交互卡片 /          │
│                                      Discord Embed              │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 出站流

出站流与入站流在上图中下半部分展示。核心流程：Agent 产出事件 → Event Bus 路由到订阅的 ChannelAdapter → `send()` 查询项目通道配置 → 翻译为 IM 消息格式 → 通过平台 API 推送。

## 4. 审批流跨通道共享

```
┌──────────────────────────────────────────────────────────────┐
│                    审批事件流（统一）                         │
│                                                              │
│  执行计划产出 → ExecutionPlanProposed 事件                   │
│                     │                                        │
│          ┌──────────┼──────────┐                             │
│          │          │          │                             │
│    SSEChannel   IMChannel  投影更新                     │
│    → Web 展示   → IM 展示      → execution_plans 表         │
│    PlanCard     交互卡片/按钮                                │
│          │          │                                        │
│    用户在 Web   用户在 IM                                    │
│    点击 Approve 点击 Approve 按钮                            │
│          │          │                                        │
│          ↓          ↓                                        │
│  ─── 同一个 ExecutionPlanApproved 事件 ───                   │
│                     │                                        │
│          ┌──────────┼──────────┐                             │
│          │          │          │                             │
│    SSEChannel   IMChannel  Agent Scheduler              │
│    → Web 更新   → IM 更新      → 调度执行 Agent              │
│                                                              │
│  关键：Hub Gateway 内所有 ChannelPlugin 共享同一套 Protocol，Event Bus 不关心事件来自哪个 Channel。                     │
│  Web Approve 和 IM Approve 产生完全相同的事件。              │
└──────────────────────────────────────────────────────────────┘
```

## 5. 消息格式翻译

Agent 产出在 Web 上有专门的卡片样式，IM 上需要降级渲染：

| 产出类型 | Web 渲染 | IM 渲染方案 |
|---------|---------|------------|
| 讨论摘要 | DiscussionPanel 卡片 | 纯文本 + 结构化列表（共识/分歧/行动项） |
| 执行计划 | PlanCard + 逐项审批 | Block Kit / 交互卡片按钮（Approve/Reject） |
| 代码 diff | OutputDiff 组件 | 截断 diff + 链接回 Web 查看完整内容 |
| 长产出 | 完整展示 | 分段推送 + "查看完整内容 → 打开 Web" |

## 6. 线程映射

IM 平台的线程概念与 Orbion 对齐：

```
Slack channel    → Orbion Project（一个项目对应一个 Slack channel）
Slack thread     → Orbion Thread（一个讨论线程对应一个 Slack thread）
企业微信群       → Orbion Project
飞书话题         → Orbion Thread
Discord channel  → Orbion Project
Discord thread   → Orbion Thread
钉钉群           → Orbion Project
```

映射配置存储在 `channel_connections` 表中，ChannelAdapter 根据配置自动路由。

## 7. ChannelConnection 数据模型

### 7.1 Pydantic 模型

```python
class ChannelConnection(BaseModel):
    """项目级 IM 通道连接配置。"""
    project_id: str            # Orbion 项目 ID
    channel_type: str          # slack/wechat/discord/...
    channel_id: str            # IM 平台的 channel/group ID
    config: dict               # 通道特定配置（bot_token, webhook_url 等）
    enabled: bool              # 是否启用
    created_at: datetime
```

### 7.2 SQL Schema

```sql
CREATE TABLE channel_connections (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id  UUID NOT NULL REFERENCES projects(id),
    channel_type VARCHAR(16) NOT NULL,
    channel_id  VARCHAR(64) NOT NULL,     -- IM 平台的 channel/group ID
    config      JSONB NOT NULL DEFAULT '{}', -- bot_token, webhook_url, etc.
    enabled     BOOLEAN NOT NULL DEFAULT true,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## 8. ChannelUserMapping 数据模型

### 8.1 Pydantic 模型

```python
class ChannelUserMapping(BaseModel):
    """IM 用户与 Orbion 用户映射。"""
    orbion_user_id: str        # Orbion 用户 ID
    channel_type: str          # slack/wechat/discord/...
    channel_user_id: str       # IM 平台用户 ID
    channel_user_name: str     # IM 平台显示名
    verified_at: datetime | None  # OAuth 验证时间
```

### 8.2 SQL Schema

```sql
CREATE TABLE channel_user_mappings (
    orbion_user_id   UUID NOT NULL REFERENCES users(id),
    channel_type     VARCHAR(16) NOT NULL,
    channel_user_id  VARCHAR(64) NOT NULL,
    channel_user_name VARCHAR(64) NOT NULL,
    verified_at      TIMESTAMPTZ NULL,
    PRIMARY KEY (orbion_user_id, channel_type, channel_user_id)
);
```

### 8.3 用户映射方案

IM 用户 ID 与 Orbion 用户 ID 的映射方案随阶段演进：

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| 手动绑定 | Orbion 用户在设置中绑定 IM 账号 | 私有化部署 |
| OAuth 映射 | IM OAuth 登录自动创建/绑定 Orbion 用户 | SaaS |
| IM 优先注册 | 通过 IM bot 直接注册 Orbion 账号 | 降低注册门槛 |

## 9. 各 IM 平台适配要点

| 平台 | 消息格式 | 审批交互 | 关键限制 | 适配器 |
|------|---------|---------|---------|--------|
| Slack | Block Kit JSON | Interactive Buttons | rate limit per method | SlackAdapter |
| 企业微信 | XML/JSON | 交互卡片 | 48 小时回复窗口 | WeChatEnterpriseAdapter |
| Discord | Embed + Markdown | Reaction emoji（👍/👎） | embed 限制 6000 字符 | DiscordAdapter |
| 钉钉 | 交互卡片 | Interactive Buttons | 企业内限定 | DingTalkAdapter |
| 飞书 | 交互卡片 | Interactive Buttons | 企业内限定 | FeishuAdapter |
| Telegram | Markdown + Inline Keyboard | Inline Buttons | 公开平台 | TelegramAdapter |
