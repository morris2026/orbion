## 1. Phase 2 定位与目标

Phase 2 在 MVP 核心循环跑通后，扩展能力边界：

- 从单服务拆分为 Hub Gateway + 4 业务服务
- 丰富 Agent 种类、审批流事件化、Agent 间协作编排
- IM Channel 接入（单通道验证 ChannelAdapter Protocol）
- 知识库 CRUD + Context Injection

## 2. Phase 2 范围裁剪

| 子系统 | Phase 2 实现 |
|--------|-------------|
| 认证 | OAuth（GitHub/Google）、密码重置 |
| EventBus | NATSEventBus 替换 InProcessEventBus |
| 服务拆分 | 从单服务拆分为 5 服务 + NATS |
| 知识库 | Wiki CRUD + Context Injection |
| 权限 | 空间级覆盖 + 资源级覆盖 + ABAC |
| Agent | 自定义 Agent、Agent 间协作编排、Skill 市场 |
| 前端 | 知识空间页面、概览/统计 |
| 部署 | SaaS 多租户支持 |
| IM Channel | 实现 + 1 个 IM 适配器（企业微信或 Slack），建表 + CRUD API |
| IM 审批交互 | 实现（Slack/企业微信） |
| IM 消息翻译 | 实现（降级渲染） |
| IM 线程映射 | 实现 |
| IM 用户映射 | 手动绑定 |
| 手机 App | 轻量版（查看/审批/通知） |

Channel 作为 Hub Gateway 内的 ChannelPlugin（SSEChannel + IMChannel），不再是独立模块。
