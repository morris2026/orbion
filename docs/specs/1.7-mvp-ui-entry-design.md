# MVP前端核心循环UI入口设计

## 背景

MVP后端22步全部完成，但前端核心循环9步中缺少6步的UI入口：创建项目、创建线程、注册Agent、添加成员、审批产出、要求修改。即使标为"有入口"的3步（发消息、请求总结、审批计划），因前置操作缺失也无法在页面上实际使用——页面是空的。

目标：补齐所有MVP核心循环的前端入口，让用户从创建项目到Git commit的每一步都能在页面上操作完成。

## 设计原则

- 主体工作在主页面完成（审批、产出操作直接内嵌在卡片上）
- "创建"类操作用Dialog触发（图标按钮+弹窗表单）
- 不改变现有三栏布局结构，只在各栏内补入口和增强现有组件
- 复用现有 `apiPost/apiGet` 和 `useWorkspace` hook

## 三栏布局设计

### 左栏：树形导航

**结构**：项目列表 + 非默认线程的两层树。

- 项目为父节点，非默认线程为子节点
- 每个项目创建时由前端自动创建默认线程（后端不自动创建），UI上不显示默认线程
- 点击项目节点 = 选中该项目的默认线程，中栏和右栏展示默认线程内容
- 额外创建的线程显示为项目的子节点

**图标按钮**（hover显示tooltip）：
- 导航栏顶部：➕新建项目
- 每个项目节点旁：➕新建线程、👤添加成员、🤖注册Agent

**新消息提示**：
- 未读线程：蓝色圆点（1条）或数字badge（多条）
- 项目节点：汇总默认线程+所有子线程的未读，有未读显示蓝色圆点
- 当前选中的项目/线程不显示提示

**交互**：
- 三栏宽度可拖拽调整（左栏160~400px，右栏200~400px）
- 左栏可折叠/展开（◀/▶按钮）

**Dialog表单**：
- 新建项目：项目名称（必填）、描述（可选）→ `apiPost('/projects', { name, description })` → 成功后自动调用 `apiPost('/projects/{newId}/threads', { title: name, type: 'discussion' })` 创建默认线程 → 自动选中新项目+默认线程
- 新建线程：线程标题（必填）→ `apiPost('/projects/{projectId}/threads', { title, type: 'discussion' })`
- 添加成员：Dialog中显示全量注册用户列表（可滚动），顶部有搜索输入框；搜索框为空时显示全量列表，搜索框有内容时调用search API替换为搜索结果；搜索失败时回退显示全量列表 → 点击用户选择 → 选择角色 → `apiPost('/projects/{id}/members', ...)`（后端需新增 `GET /auth/users/search?username=xxx` 和 `GET /auth/users` 全量列表）
- 注册Agent：勾选Agent类型（summary/decompose/execute）→ `apiPost('/projects/{projectId}/agents', { agent_type, model_id, display_name })`

### 中栏：IM泡泡讨论

**上下分区**：消息区域（上）+ 输入区域（下），中间水平分隔条可拖拽调整比例。

**消息样式**：IM泡泡风格，与常见IM保持一致：
- 自己的消息：右对齐，蓝色泡泡，头像在右侧，名字和时间在泡泡上方右对齐（时间→名字）
- 别人的消息：左对齐，浅色泡泡，头像在左侧，名字和时间在泡泡上方左对齐（名字→时间）
- Agent总结消息：左对齐，浅蓝泡泡带蓝色边框，头像🤖图标，名字和时间左对齐
- 泡泡宽度自适应，max-width 70%
- 头像为圆形首字母或Agent图标
- 时间戳格式：今天→HH:mm，昨天→"昨天 HH:mm"，更早→YYYY-MM-DD HH:mm

**输入区域**：
- textarea + 发送按钮，只有一个发送按钮
- 请求总结等操作用斜杠命令：`/summarize` 请求总结、`/help` 查看所有命令
- placeholder提示："输入消息，或 /summarize 请求总结..."

**发送逻辑**：
- SSE单一来源：POST仅触发后端发布事件，前端通过SSE回传显示消息，不做乐观更新
- 斜杠命令解析：发送前检测 `/summarize` 等前缀，匹配则设置 `request_summary: true`；纯 `/summarize`（无附加文字）时 content 发送固定文案"请总结当前讨论要点"；`/summarize 观点` 时 content 为"观点"

### 右栏：三Tab执行面板

**三个Tab**：计划 / 产出 / Agent，顶部按钮切换。

**计划Tab**：
- 按状态分组：待审批（proposed）、已批准（approved）、已拒绝（rejected）
- proposed计划卡片：显示task列表 + 批准/拒绝按钮
- 批准操作：调用 `apiPost('/plans/{id}/approve', { approved_tasks })`（MVP裁剪：不实现 modifications 字段的部分审批修改功能）
- 拒绝操作：卡片内展开reason输入框，调用 `apiPost('/plans/{id}/reject', { reason, suggestions })`

**产出Tab**：
- 按状态分组：待审批（generated）、待修改（revision_requested）、已批准（approved）
- generated产出卡片：diff预览 + 批准产出/要求修改按钮
- 批准产出：调用 `apiPost('/outputs/{id}/approve', { feedback })`
- 要求修改：卡片内展开输入区（问题描述+改进建议），调用 `apiPost('/outputs/{id}/request-revision', { issues, suggestions })`
- revision_requested产出卡片：显示修改要求内容
- approved产出卡片：显示"已自动commit → git log可查"

**Agent Tab**：
- 项目下所有Agent卡片，显示Agent类型、运行状态（idle/running/错误，前端UI文案映射：running→"工作中"）、最近活动
- running状态卡片用蓝色边框，idle用绿色/灰色

## 后端新增

| API | 说明 |
|-----|------|
| `GET /auth/users` | 全量注册用户列表（返回 user_id, username, display_name, status, created_at） |
| `GET /auth/users/search?username=xxx` | 搜索匹配用户（返回 user_id, username, display_name, status, created_at），前端用于动态过滤列表 |
