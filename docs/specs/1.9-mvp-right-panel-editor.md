# 1.9 右栏文件编辑器设计

## 概述

右栏从当前的扁平 `ExecutionPanel` 改为 Tab 容器，首期实现「文件」Tab，为后续「计划」「产出」「Agent」Tab 预留接口。文件 Tab 内部采用 VSCode 式三层嵌套布局，支持文件浏览、代码编辑、Markdown 预览、Source Control 基本操作。

## 技术选型

| 组件 | 方案 | 理由 |
|------|------|------|
| 代码编辑器 | `@monaco-editor/react` + `monaco-editor` | VSCode 同款引擎，内置 DiffEditor，多模型支持天然适配单文件切换 |
| 文件树 | `react-arborist` | 明确对标 VSCode sidebar 设计，虚拟化渲染，支持 Tailwind 自定义样式 |
| 分栏布局 | `react-resizable-panels`（已安装） | 编辑+预览左右分栏、侧边栏调宽 |
| Markdown 预览 | `react-markdown` + `remark-gfm` | React 生态标准方案，轻量可扩展 |

## 布局结构

```
右栏
├── Tab 栏（文件 | 计划 | 产出 | Agent）
└── 文件 Tab
    ├── 活动栏（48px 图标列：文件浏览器 / Source Control）
    ├── 侧边栏（可折叠，240px 默认宽度）
    │   ├── Explorer 面板（文件树）
    │   └── Source Control 面板
    │       ├── 上栏：仓库列表（可折叠、可拖拽调高、单选）
    │       └── 下栏：选中仓库状态（Staged/Changes + Commit）
    └── 主区域
        ├── 文件 Tab 栏
        ├── 编辑区工具栏（保存按钮 + 预览开关）
        ├── 编辑区（Monaco Editor）
        ├── 预览分隔条
        └── 预览区（右侧，可关闭/打开，Markdown 渲染）
```

活动栏图标切换侧边栏内容。侧边栏可折叠。主区域编辑和预览左右分栏，预览可关闭/打开。

## 前端组件

| 组件 | 职责 |
|------|------|
| `RightPanelTabs` | 右栏 Tab 容器，管理 Tab 切换，为后续 Tab 预留接口 |
| `FileTab` | 文件 Tab 主容器，管理活动栏/侧边栏/主区域的布局 |
| `ActivityBar` | 活动栏，图标切换 Explorer / Source Control |
| `ExplorerPanel` | 文件浏览器，基于 react-arborist 的文件树 |
| `SourceControlPanel` | Source Control 侧边栏，上下分栏 |
| `RepoList` | 上栏：仓库列表（可折叠、可拖拽调高、单选） |
| `RepoStatus` | 下栏：选中仓库的 Staged/Changes 文件列表 + stage/unstage + Commit 输入 |
| `FileEditor` | 主区域：文件 Tab 栏 + Monaco Editor + 保存按钮 + 预览开关；`monacoError` prop 控制降级为纯文本 textarea |
| `FilePreview` | 预览区（Markdown 渲染），右侧可关闭面板，可扩展渲染接口 |

## API 设计

### 文件浏览 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/projects/{project_id}/repos/{repo_name}/tree` | GET | 获取目录树，`?path=` 指定子目录 |
| `/projects/{project_id}/repos/{repo_name}/files` | GET | 读取文件内容，`?path=` 指定文件路径，`?ref=HEAD` 读取 git HEAD 版本（用于 DiffEditor original） |
| `/projects/{project_id}/repos/{repo_name}/files` | PUT | 保存文件内容，`?path=` 指定文件路径 |

### 仓库管理 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/projects/{project_id}/repos` | GET | 列出项目仓库 |
| `/projects/{project_id}/repos` | POST | 添加仓库（URL 或目录名） |
| `/projects/{project_id}/repos/{repo_name}` | DELETE | 删除仓库（删除物理目录） |

### Source Control API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/projects/{project_id}/repos/{repo_name}/status` | GET | git status（Staged/Changes 文件列表 + 状态标识） |
| `/projects/{project_id}/repos/{repo_name}/stage` | POST | stage 文件，body: `{ paths: string[] }` |
| `/projects/{project_id}/repos/{repo_name}/unstage` | POST | unstage 文件，body: `{ paths: string[] }` |
| `/projects/{project_id}/repos/{repo_name}/commit` | POST | commit，body: `{ message: string }` |

## 数据流

1. 选择项目 → 加载仓库列表（`GET /repos`） → 默认选中第一个仓库 → 加载文件树（`GET /tree`）+ git status（`GET /status`）
2. 切换仓库 → 重新加载文件树 + git status
3. Explorer 点击文件 → 加载文件内容到 Monaco（`GET /files`） → 如果是 Markdown 且预览打开则渲染
4. Source Control 点击文件 → 主区域切换为 Monaco DiffEditor（original=HEAD 版本通过 `GET /files?ref=HEAD` 获取，modified=工作区版本）
5. 编辑 → Ctrl+S / 保存按钮 → PUT 写入文件系统 → 刷新 git status
6. stage/unstage/commit → 调用对应 API → 刷新 git status

## 文件状态与编辑交互

- 单文件编辑，点击即切换
- 编辑过的文件在 Tab 栏标记修改标识（圆点），保存后清除
- 保存：工具栏保存按钮 + Ctrl+S 快捷键 → PUT 写入 → 刷新 git status
- 预览：仅 Markdown 类型显示「预览」开关；打开后右侧渲染，编辑变更 debounce 300ms 实时反映；渲染器可扩展
- Diff 查看：Source Control 点击文件 → Monaco DiffEditor；切换回 Explorer 点击文件 → 恢复正常编辑
- Monaco 降级：FileTab 层通过 `loader.init().catch()` + 超时兜底检测加载失败，将 `monacoError` 状态传入 FileEditor；FileEditor 收到后渲染纯文本 textarea 替代 Monaco，保留编辑和保存能力

## 后端实现要点

### 仓库管理

- 仓库列表来自自动扫描项目目录下的 git 仓库（发现 `.git` 即识别为仓库）
- 添加仓库：用户指定 URL（git clone 到项目目录下）或目录名（新建目录并 git init）；若仓库根目录下已存在同名目录则返回 400 失败
- 删除仓库：删除物理目录，扫描自动移除
- 文件浏览和 Source Control 的仓库列表统一来自扫描结果

### 文件操作

- 文件树读取：遍历 `project_repo_path(project_id, repo_name)` 下的目录结构，排除 `.git`
- 文件读取/写入：直接操作文件系统，路径必须 `is_relative_to` 仓库根目录（防路径穿越）
- 保存文件后不自动 git commit

### Git 操作

- 扩展现有 `GitService`，新增 `status`/`stage`/`unstage`/`commit` 方法，按 `project_id + repo_name` 隔离
- `status` 返回分组的文件列表：staged（index 中有变更）+ changes（工作区有变更但未 stage）
- stage/unstage 接收文件路径列表
- commit 接收 message，仅 commit 已 staged 的文件

## 错误处理

| 场景 | 处理 |
|------|------|
| 文件操作越界（路径穿越） | 403 |
| 文件/目录不存在 | 404 |
| 仓库不存在（.git 消失） | 404 |
| 仓库添加失败（同名目录已存在、clone 失败、目录无效） | 400 + 错误信息 |
| 并发写入同一文件 | 后写入覆盖，不做锁（MVP 单用户为主） |
| Monaco 加载失败 | FileEditor 通过 `monacoError` prop 接收降级状态，渲染纯文本 textarea；检测由 FileTab 层通过 `loader.init().catch()` + 超时兜底完成 |
