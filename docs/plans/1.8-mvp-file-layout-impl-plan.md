# Orbion 文件系统工作目录 实施计划

## 策略

**配置层先行 + 逐层向上适配**：先改 Settings 配置层（单根目录 + 派生路径），再适配依赖配置的三个消费者（AgentMemory、GitService、ProjectService），最后组装到 main.py 并验证启动流程。

选择理由：
- Settings 是所有消费者的硬依赖，必须先改
- 三个消费者互相独立，但都依赖 Settings 的派生路径方法
- main.py 组装放在最后，确保所有模块接口稳定后再接入
- Agent 模板机制依赖 Settings + AgentMemory，放在消费者之后

## 约束

1. 每步大小合适，可独立验证
2. 步骤依赖关系正确
3. 严格符合 1.8 设计文档目标，不缩水不遗漏
4. 每步遵循 TDD 流程（写测试→验证失败→实现→验证通过→commit）
5. 计划不含实现细节（无代码、无函数签名、无 SQL）
6. 每步有总结好的 commit message
7. 完备测试覆盖：UT + 集成测试

## 步骤总览

共 7 步：

| 步骤 | 核心增量 | 对应验证点 |
|------|---------|-----------|
| 1 | Settings 重构：root_dir 替代双路径 + 派生路径方法 | MVP-FL-1.1–MVP-FL-1.8 |
| 2 | AgentMemory 适配新路径：三层记忆 + Settings 注入 | MVP-FL-2.1–MVP-FL-2.7 |

### - [x] 步骤 1：Settings 重构 — root_dir 替代双路径

**增量**：OrbionConfigSchema 移除 repo_path 和 memory_base_path，替换为 root_dir；Settings 类同步替换并添加全部派生路径方法（projects_dir、platform_memory_path、project_dir、project_memory_path、agent_memory_path、project_repo_path、thread_dir、output_payload_path）；其中 thread_dir 和 output_payload_path 为接口预留（MVP 不调用，不创建目录）；更新 orbion.json；更新现有 test_config.py 中的旧断言

**测试设计**：见 [1.8-mvp-file-layout-test-design](1.8-mvp-file-layout-test-design.md) MVP-FL-1.1–MVP-FL-1.8

### - [x] 步骤 2：AgentMemory 适配新路径结构

**注意**：本步修改 AgentMemory 构造签名后，需同步修改 main.py 中对应的初始化代码（仅接口适配，不改变启动流程逻辑）
