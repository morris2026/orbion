# 1.9 右栏文件编辑器 实施计划

## 关联文档

- **设计规格**：[1.9-mvp-right-panel-editor.md](../specs/1.9-mvp-right-panel-editor.md) — 右栏 Tab 容器、文件编辑器、Source Control 的完整设计定义
- **测试设计**：[1.9-mvp-right-panel-editor-test-design.md](1.9-mvp-right-panel-editor-test-design.md) — 每步的 TC 编号对应用例的详细描述和验证标准

## 策略

**后端先行 + 前端基础设施 + 组件迭代**：先补后端仓库管理和文件操作 API，再安装前端编辑器依赖和基础设施，然后按活动栏→侧边栏→主区域的顺序迭代实现文件 Tab，最后集成到 Workspace 右栏。

选择理由：
- 后端 API 是前端的数据依赖，必须先有
- Monaco + react-arborist 等依赖安装后才能开发前端组件
- 活动栏和侧边栏是主区域的数据来源，先实现
- 主区域的编辑/预览依赖侧边栏的文件选择事件
- 最后替换 Workspace 右栏，确保集成正确

## 约束

1. 每步大小合适，可独立验证
2. 步骤依赖关系正确，标注可并行的步骤对
3. 严格符合 1.9 设计文档目标，不缩水不遗漏
4. 每步遵循 TDD 流程（写测试→验证失败→实现→验证通过→commit）
5. 计划不含实现细节（无代码、无函数签名、无 SQL）
6. 每步有总结好的 commit message
7. 完备测试覆盖：UT + 集成测试 + 前端组件测试

## 第 1 节：后端仓库与文件 API（3 步，步骤 1-2 可并行）

### - [x] 步骤 1：仓库管理 API

**增量**：新增 `app/biz/repos/` 模块（service + routes + models），实现仓库自动扫描、添加（URL clone / 目录名 git init，同名目录已存在返回 400）、删除（删除物理目录）三个 API 端点；路由挂载在 `/projects/{project_id}/repos` 下；权限校验复用现有成员检查模式

**依赖**：无（复用现有 Settings.project_repo_path + GitService）

**测试设计**：见 [1.9-mvp-right-panel-editor-test-design](1.9-mvp-right-panel-editor-test-design.md) MVP-RE-1.1–MVP-RE-1.8

**commit message**：`feat: 仓库管理 API（扫描/添加/删除）`

---

### - [x] 步骤 2：文件操作 API

**增量**：新增 `app/biz/files/` 模块（service + routes + models），实现文件树遍历（排除 .git）、文件读取（支持 `?ref=HEAD` 参数读取 git HEAD 版本，用于 DiffEditor）、文件保存三个 API 端点；路径安全校验（is_relative_to 仓库根目录，防路径穿越）；文件树返回扁平列表（path + type + name）；路由挂载在 `/projects/{project_id}/repos/{repo_name}/tree` 和 `/projects/{project_id}/repos/{repo_name}/files` 下

**依赖**：步骤 1（仓库扫描 API 提供仓库列表，文件操作需要仓库路径）

**注意**：`?ref=HEAD` 读取需调用 git show HEAD:path 获取已提交版本，无 commit 的仓库返回空内容

**测试设计**：见 [1.9-mvp-right-panel-editor-test-design](1.9-mvp-right-panel-editor-test-design.md) MVP-RE-2.1–MVP-RE-2.9

**commit message**：`feat: 文件浏览与编辑 API（目录树/读取/保存）`

---

### - [x] 步骤 3：Git status/stage/unstage/commit API

**增量**：扩展现有 `app/biz/git/service.py`，新增 `status`（返回 staged/changes 分组文件列表）、`stage`（接收文件路径列表，git add）、`unstage`（接收文件路径列表，git reset HEAD）、`commit`（接收 message，git commit）四个方法；新增 `app/biz/git/routes.py` 对应端点；均按 `project_id + repo_name` 隔离

**依赖**：步骤 1（仓库扫描确认仓库存在）

**测试设计**：见 [1.9-mvp-right-panel-editor-test-design](1.9-mvp-right-panel-editor-test-design.md) MVP-RE-3.1–MVP-RE-3.8

**commit message**：`feat: Source Control API（status/stage/unstage/commit）`

## 第 3 节：前端组件（4 步，步骤 5-6 可并行）

### - [x] 步骤 5：ActivityBar + ExplorerPanel

**增量**：新增 `ActivityBar` 组件（活动栏图标列，切换 Explorer / Source Control）；新增 `ExplorerPanel` 组件（基于 react-arborist 的文件树，点击文件触发 onFileSelect 回调，切换项目重新加载文件树）；新增 `useFileTab` hook（管理仓库列表、文件树数据、选中仓库/文件、文件内容、git status 等 FileTab 全部状态，封装 API 调用）

**依赖**：步骤 4（react-arborist + 类型定义）、步骤 2（文件树 API）、步骤 1（仓库列表 API）

**测试设计**：见 [1.9-mvp-right-panel-editor-test-design](1.9-mvp-right-panel-editor-test-design.md) MVP-RE-5.1–MVP-RE-5.8

**commit message**：`feat: 活动栏+文件浏览器+useFileTab hook`

**依赖**：步骤 4（Monaco + react-markdown）、步骤 5（onFileSelect 回调 + useFileTab 状态）

**测试设计**：见 [1.9-mvp-right-panel-editor-test-design](1.9-mvp-right-panel-editor-test-design.md) MVP-RE-6.1–MVP-RE-6.11

**增量**：新增 `SourceControlPanel` 组件（上下分栏：上栏 RepoList 仓库列表可折叠可拖拽调高单选，下栏 RepoStatus 选中仓库的 Staged/Changes 分组 + stage/unstage 按钮 + Commit 输入框和提交按钮）；新增 `RepoList` 组件（仓库列表，点击切换选中仓库，折叠/展开，拖拽调高）；新增 `RepoStatus` 组件（Staged/Changes 文件列表，点击文件触发 onFileSelect 并切换 DiffEditor 模式，stage/unstage 操作按钮，Commit 输入+提交）
