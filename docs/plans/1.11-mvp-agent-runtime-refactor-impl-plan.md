# Orbion Agent Runtime 重构 实施计划

## 策略

**数据模型与加密底座先行，逐层向上构建 AgentRuntimeAdapter → SkillPermissionChecker → AgentRuntime → 事件调度 → 失败恢复 → Critic → 审批传播 → 可观测性**：先建全部新表 + AES-GCM 加密 + 用户模型配置 API，再实现 AgentRuntimeAdapter（封装 Agent SDK，OpenAI 端先实现 + Claude 端预留）与 SkillPermissionChecker（SDK 无关的业务策略层），然后是 AgentRuntime 核心 dispatch/chat/lightweight_call，接着事件驱动调度、失败恢复、Critic、审批与变更传播，最后是 token 统计、可观测性与运维收尾。

选择理由：

- 加密底座（AES-GCM）和 UserModel 表是 AdapterFactory 的硬依赖，必须先就绪
- AgentRuntimeAdapter 是 dispatch 的核心依赖（提供 `run_streamed` + `complete` 两组接口），要在 AgentRuntime 之前实现
- SkillPermissionChecker 作为 SDK hook 注入 Adapter，需先于 AgentRuntime
- AgentRuntime.dispatch 是事件消费方，必须在 EventBus/scheduler 路由之前
- 失败恢复（reconcile/事务补偿/优雅停机）依赖 agent_runs 表和 dispatch 流程
- Critic 是 task completed 后的可选触发，依赖 dispatch + agent_runs.run_kind=critic
- 审批合并依赖 artifact 状态机 + WorktreeService.merge（已实现）+ 变更传播依赖 based_on_artifacts 查询
- 可观测性与 token 统计横切所有模块，放最后接入避免反复改动

## 约束

1. 每步大小合适，可独立验证
2. 步骤依赖关系正确
3. 严格符合 agent-runtime-refactor-design 设计文档目标，不缩水不遗漏
4. 每步遵循 TDD 流程（写测试→验证失败→实现→验证通过→commit）
5. 计划不含实现细节（无代码、无函数签名、无 SQL）
6. 每步有总结好的 commit message
7. 完备测试覆盖：UT + 集成测试 + 关键路径 E2E
8. 重构性质与切换时点：项目尚未发布，无生产数据需要保护，旧 `app/biz/agents/{runtime,service,scheduler}` 与 `execution_plans`/`task_outputs` 表可直接替换，无需数据迁移与双写过渡。按三个时点分阶段切换：
   - **切换点 A（步骤 14 完成后）**：API 路由层切到新 `AgentRuntime` 入口——`POST /projects/{id}/agents/{type}/chat`（§12.4）+ 轻量调用（§12.5）+ §12.7 运行控制端点全部就绪；删除旧 `app/biz/agents/routes.py`（整文件，dispatch/chat/轻量/运行控制路由完全切到新入口）。`main.py` 同步移除旧 `agent_router` 注册。§12.3 意图发布端点在步骤 16 新增（不依赖旧 routes.py）。旧 `service.py` / `runtime.py` / `declarations.py` 等仍存活（被 `main.py` lifespan 实例化），到切换点 C 统一清理
   - **切换点 B（步骤 16 完成后）**：`AgentScheduler` 与 `EventBus` 完全切到新 outbox 模型；删除旧 `app/biz/agents/scheduler.py`；新 `agent_runs`/`tasks`/`artifacts` 表成为唯一权威
   - **切换点 C（步骤 22 完成后）**：DROP `execution_plans`/`task_outputs` 表（迁移脚本 `DROP TABLE`）+ 删除 `app/biz/plans/` 与 `app/biz/outputs/` 模块 + 删除旧 `app/biz/agents/` 整目录残留（`__init__.py` + `service.py` / `runtime.py` / `declarations.py` / `memory.py` / `templates.py` / `skills.py` / `models.py` + `adapters/` 整目录含 `_legacy/`，`routes.py` / `scheduler.py` 已分别在切换点 A/B 删除）
   - 每个切换点单独 commit（如 `refactor: dispatch 路径切到新 AgentRuntime`），不与功能 commit 混杂
   - 切换点 C 前确认 `make check` 全部通过 + E2E-1 完整工作流在 staging 跑通
