# 1.9 右栏文件编辑器 测试设计

本文档为 [1.9-mvp-right-panel-editor-impl-plan](1.9-mvp-right-panel-editor-impl-plan.md) 的配套测试设计，按实施计划步骤顺序组织。每个步骤的验证标准在此展开为具体用例划分、主体流程和检查项。

## 用例格式

每个用例包含：
- **用例名**：概括测试意图
- **类型**：UT / API / 集成 / Vitest / E2E
- **主体流程**：操作序列
- **检查项**：断言要点

## 步骤 1：仓库管理 API

### MVP-RE-1.1 扫描项目下的 git 仓库

- **类型**：UT
- **主体流程**：在项目目录下创建两个子目录并 git init（repo_a、repo_b），调用仓库扫描方法
- **检查项**：返回列表包含 "repo_a" 和 "repo_b"

### MVP-RE-1.2 扫描排除非 git 目录

- **类型**：UT
- **主体流程**：在项目目录下创建普通目录（no_git）和 git 仓库（has_git），调用仓库扫描方法
- **检查项**：返回列表仅包含 "has_git"，不含 "no_git"

### MVP-RE-1.3 添加仓库 — URL（git clone）

- **类型**：集成
- **主体流程**：POST /projects/{id}/repos，body 包含 url 和 repo_name，仓库根目录下不存在同名目录
- **检查项**：仓库目录被创建，包含 .git；仓库扫描结果包含新仓库

### MVP-RE-1.4 添加仓库 — 目录名（git init）

- **类型**：集成
- **主体流程**：POST /projects/{id}/repos，body 包含 repo_name（无 url），仓库根目录下不存在同名目录
- **检查项**：仓库目录被创建，包含 .git；仓库扫描结果包含新仓库

### MVP-RE-1.5 添加仓库 — 同名目录已存在

- **类型**：集成
- **主体流程**：项目 repo 目录下已存在同名目录（非 git 仓库），POST /projects/{id}/repos 添加同名仓库
- **检查项**：返回 400，错误信息说明同名目录已存在

### MVP-RE-1.5a 添加仓库 — clone 失败

- **类型**：集成
- **主体流程**：POST /projects/{id}/repos，body 包含无效 url（如 `https://nonexistent.invalid/repo.git`）
- **检查项**：返回 400，错误信息包含 clone 失败原因

### MVP-RE-1.5b 添加仓库 — 目录名无效

- **类型**：集成
- **主体流程**：POST /projects/{id}/repos，body 包含含路径分隔符的目录名（如 `../evil`）
- **检查项**：返回 400（路径校验拒绝）

### MVP-RE-1.6 删除仓库

- **类型**：集成
- **主体流程**：仓库已存在，DELETE /projects/{id}/repos/{repo_name}
- **检查项**：仓库物理目录被删除；仓库扫描结果不再包含该仓库

### MVP-RE-1.7 非项目成员操作

- **类型**：集成
- **主体流程**：非项目成员调用仓库管理 API（添加/删除/列表）
- **检查项**：返回 403

### MVP-RE-1.8 仓库不存在时删除

- **类型**：集成
- **主体流程**：DELETE /projects/{id}/repos/{nonexistent}
- **检查项**：返回 404

## 步骤 2：文件操作 API

### MVP-RE-2.1 获取文件树

- **类型**：UT
- **主体流程**：仓库目录下有嵌套结构（src/main.ts、docs/guide.md、README.md），调用文件树遍历
- **检查项**：返回扁平列表，每项包含 path、type（file/dir）、name；排除 .git 目录

### MVP-RE-2.2 获取文件树 — 空仓库

- **类型**：UT
- **主体流程**：仓库目录为空（只有 .git），调用文件树遍历
- **检查项**：返回空列表

### MVP-RE-2.3 读取文件内容

- **类型**：集成
- **主体流程**：仓库中有 src/main.ts，GET /projects/{id}/repos/{repo}/files?path=src/main.ts
- **检查项**：返回文件内容字符串

### MVP-RE-2.3a 读取文件 HEAD 版本

- **类型**：集成
- **主体流程**：仓库中有已 commit 的 src/main.ts（内容 "v1"），工作区内容已改为 "v2"，GET /projects/{id}/repos/{repo}/files?path=src/main.ts&ref=HEAD
- **检查项**：返回 "v1"（HEAD 版本）

### MVP-RE-2.3b 读取 HEAD 版本 — 无 commit 的仓库

- **类型**：集成
- **主体流程**：新建仓库（无任何 commit），GET /projects/{id}/repos/{repo}/files?path=any.ts&ref=HEAD
- **检查项**：返回空内容或 404

### MVP-RE-2.4 读取不存在的文件

- **类型**：集成
- **主体流程**：GET /projects/{id}/repos/{repo}/files?path=nonexistent.ts
- **检查项**：返回 404

### MVP-RE-2.5 保存文件内容

- **类型**：集成
- **主体流程**：PUT /projects/{id}/repos/{repo}/files?path=src/main.ts，body 包含新内容
- **检查项**：文件内容更新为新内容

### MVP-RE-2.6 保存新建文件

- **类型**：集成
- **主体流程**：PUT /projects/{id}/repos/{repo}/files?path=new-file.ts，body 包含内容
- **检查项**：新文件创建，内容正确；中间目录自动创建

### MVP-RE-2.7 路径穿越攻击

- **类型**：集成
- **主体流程**：GET /projects/{id}/repos/{repo}/files?path=../../../etc/passwd
- **检查项**：返回 403（路径越界）

### MVP-RE-2.8 非项目成员访问

- **类型**：集成
- **主体流程**：非项目成员调用文件操作 API
- **检查项**：返回 403

### MVP-RE-2.9 并发写入 — 后写入覆盖

- **类型**：UT
- **主体流程**：两个并发 PUT 请求写入同一文件（内容 A 和内容 B），均成功
- **检查项**：文件最终内容为其中之一（后写入覆盖），不抛异常

## 步骤 3：Git status/stage/unstage/commit API

### MVP-RE-3.1 git status — 有变更文件

- **类型**：UT
- **主体流程**：仓库中有已修改但未 stage 的文件和已 stage 的文件，调用 status 方法
- **检查项**：返回分组的文件列表：staged 列表含已 stage 文件（含状态标识 M/A/D），changes 列表含未 stage 文件（含状态标识 M/A/D）

### MVP-RE-3.2 git status — 无变更

- **类型**：UT
- **主体流程**：仓库无变更，调用 status 方法
- **检查项**：staged 和 changes 均为空列表

### MVP-RE-3.3 stage 文件

- **类型**：集成
- **主体流程**：文件有未 stage 的修改，POST /projects/{id}/repos/{repo}/stage，body 包含文件路径
- **检查项**：status 中该文件从 changes 移到 staged

### MVP-RE-3.4 unstage 文件

- **类型**：集成
- **主体流程**：文件已 stage，POST /projects/{id}/repos/{repo}/unstage，body 包含文件路径
- **检查项**：status 中该文件从 staged 移到 changes

### MVP-RE-3.5 commit

- **类型**：集成
- **主体流程**：文件已 stage，POST /projects/{id}/repos/{repo}/commit，body 包含 message
- **检查项**：commit 成功；status 中 staged 列表清空；get_recent_commits 包含新 commit

### MVP-RE-3.6 commit 无 staged 文件

- **类型**：集成
- **主体流程**：无 staged 文件，POST /projects/{id}/repos/{repo}/commit
- **检查项**：返回 400（无可提交的变更）

### MVP-RE-3.7 仓库不存在

- **类型**：集成
- **主体流程**：对不存在的仓库调用 status/stage/unstage/commit
- **检查项**：返回 404

### MVP-RE-3.8 非项目成员操作

- **类型**：集成
- **主体流程**：非项目成员调用 Source Control API
- **检查项**：返回 403
