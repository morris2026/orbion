# MVP前端核心循环UI入口 实施计划

## 关联文档

- **设计规格**：[1.7-mvp-ui-entry-design.md](../specs/1.7-mvp-ui-entry-design.md) — 三栏布局、组件交互、API调用、斜杠命令的完整设计定义
- **测试设计**：[1.1-mvp-ui-entry-test-design.md](1.1-mvp-ui-entry-test-design.md) — 每步的TC编号对应用例的详细描述和验证标准

## 策略

**后端先行 + 基础设施 + 三栏迭代**：先补后端缺失的用户搜索API，再安装前端基础设施（shadcn组件、类型定义、hook扩展），然后按左栏→中栏→右栏的顺序迭代实现各栏增强，最后集成Workspace页面并更新测试。

## 约束

1. 每步大小合适，可独立验证
2. 步骤依赖关系正确，标注可并行的步骤对
3. 严格符合设计文档目标，不缩水不遗漏
4. 每步遵循TDD流程（写测试→验证失败→实现→验证通过→commit）
5. 计划不含实现细节（无代码、无函数签名、无SQL）
6. 每步有总结好的commit message
